# R2

## Import

```python
from xampler.r2 import R2Bucket, R2Conditional, R2HttpMetadata, R2Range
```

## Copy this API

```python
bucket = R2Bucket(env.BUCKET)
obj = bucket.object("notes/a.txt")

await obj.write_text("hello")
text = await obj.read_text()
info = await obj.stat()

stream = await obj.byte_stream()
async for chunk in stream.iter_bytes():
    ...
```

## Main classes

- `R2Bucket(raw)` wraps an R2 bucket binding.
- `R2ObjectRef` is a cheap handle for one key.
- `R2Object` wraps a fetched object with `text()`, `json()`, `bytes()`, and `chunks()`.
- `R2Range` models byte ranges.
- `R2Conditional` models ETag/date preconditions.
- `R2MultipartUpload` supports `async with`, `upload_part()`, `complete()`, and `abort()`.

## Common methods

```python
await bucket.put_text(key, value)
await bucket.get_text(key)
await bucket.put_bytes(key, data)
await bucket.get_bytes(key, byte_range=R2Range(offset=0, length=100))
await bucket.exists(key)
await bucket.delete(key)
await bucket.list(prefix="logs/")
async for item in bucket.iter_objects(prefix="logs/"):
    ...
```

## Testability

Use a fake bucket binding with `put`, `get`, `head`, `delete`, and `list`. Keep large-body behavior behind `byte_stream()`/`iter_bytes()` so tests can provide small async byte iterators.

## Runtime notes

Small helpers buffer into Python. For large objects, prefer streams and avoid unnecessary JS↔Python copies. `.raw` remains available for R2 features not yet modeled.
