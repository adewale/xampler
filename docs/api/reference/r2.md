# R2

## Import

```python
from xampler.r2 import R2Bucket, R2Conditional, R2HttpMetadata, R2Range
```

## Copy this API

```python
bucket = R2Bucket(env.BUCKET)
obj = bucket.object("notes/a.txt")

await obj.write_text("hello")
text = await obj.read_text()
info = await obj.stat()

stream = await obj.byte_stream()
async for chunk in stream.iter_bytes():
    ...
```

## Main classes

- `R2Bucket(raw)` wraps an R2 bucket binding.
- `R2ObjectRef` is a cheap handle for one key.
- `R2Object` wraps a fetched object with `text()`, `json()`, `bytes()`, and `chunks()`.
- `R2Range` models byte ranges.
- `R2Conditional` models ETag/date preconditions.
- `R2MultipartUpload` supports `async with`, `upload_part()`, `complete()`, and `abort()`.

## Common methods

```python
await bucket.put_text(key, value)
await bucket.get_text(key)
await bucket.put_bytes(key, data)
await bucket.get_bytes(key, byte_range=R2Range(offset=0, length=100))
await bucket.exists(key)
await bucket.delete(key)
await bucket.list(prefix="logs/")
async for item in bucket.iter_objects(prefix="logs/"):
    ...
```

## Capability table

| Operation | Status | Notes |
|---|---|---|
| Text/JSON/bytes write and read | Supported | Local verifier writes text, uploads a JPEG, streams it back, and byte-compares it. |
| `head`, `exists`, `delete` | Supported | Missing reads return `None`; `exists` returns `False`. |
| `list`, `iter_objects` | Supported | Cursor shape is exposed; local verifier covers listing. |
| Multipart upload | Caveated | API shape exists and uses `async with`; deeper completion/abort and remote tests remain pending. |
| Public object URL | Caveated | Requires a public bucket, `r2.dev`, or custom domain configured outside this binding wrapper. |
| Signed download/upload URL | Unsupported / throws | A Workers R2 binding cannot sign URLs. Use S3-compatible HTTP credentials or a future explicit signing helper. |
| CORS, lifecycle, event notifications | Not covered | Configure through Wrangler/Cloudflare APIs or use `.raw` where appropriate. |

## Public URLs and signed URLs

An R2 binding is excellent for in-Worker reads and writes, but it is not a URL signer. Do not infer a public URL from an object key.

Use a public URL only when the bucket/domain is deliberately public:

```python
public_url = f"https://assets.example.com/{key}"
```

For private browser downloads or direct browser uploads, the safe design is an explicit signing path backed by S3-compatible R2 HTTP credentials. The current `xampler.r2` binding wrapper does **not** provide this. A future helper should fail loudly when credentials are absent rather than silently returning an unsafe URL.

## User-upload safety

When serving user uploads, be explicit about trust boundaries:

- Treat stored `content_type` as user-controlled unless your application set it.
- HTML/SVG served inline from your domain can become stored XSS.
- Prefer `Content-Disposition: attachment` for untrusted downloadable files.
- Direct browser uploads should have short expiries and server-side size limits. A presigned PUT without a size cap can become a storage-cost DoS vector.
- CORS is bucket/platform configuration, not something this Python wrapper can safely infer.

## Streaming and memory

Small helpers such as `read_text()`, `read_json()`, `read_bytes()`, and `R2Object.bytes()` buffer into Python memory. For large objects, prefer streaming paths:

```python
async for chunk in bucket.iter_bytes("large.bin"):
    ...

stream = await bucket.object("large.bin").byte_stream()
async for line in stream.iter_lines():
    ...
```

`put_stream()` accepts a JavaScript stream and keeps the large-body path on the runtime side instead of copying through Python memory.

## Errors and absence

Expected missing objects are Pythonic:

```python
assert await bucket.get_text("missing.txt") is None
assert await bucket.exists("missing.txt") is False
```

Auth, precondition, unsupported-feature, and provider failures should be treated as exceptional. Shared code can use `xampler.errors.XamplerError` for normalized route/tool/CLI error payloads.

## Testability

Use a fake bucket binding with `put`, `get`, `head`, `delete`, and `list`. Keep large-body behavior behind `byte_stream()`/`iter_bytes()` so tests can provide small async byte iterators.

## Runtime notes

Small helpers buffer into Python. For large objects, prefer streams and avoid unnecessary JS↔Python copies. `.raw` remains available for R2 features not yet modeled.
