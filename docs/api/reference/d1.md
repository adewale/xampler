# D1

## Import

```python
from xampler.d1 import D1Database, D1Statement
```

## Copy this API

```python
db = D1Database(env.DB)
rows = await db.query("SELECT quote, author FROM quotes LIMIT 10")
row = await db.query_one("SELECT quote FROM quotes WHERE author = ?", "PEP 20")
```

## Main classes

- `D1Database(raw)` wraps a D1 binding.
- `D1Statement(raw_statement)` wraps a prepared statement.

## Common methods

```python
statement = db.statement("SELECT * FROM quotes WHERE author = ?")
rows = await statement.all("PEP 20")
row = await statement.one("PEP 20")
quote = await statement.one_as(Quote, "PEP 20")

await db.execute("CREATE TABLE IF NOT EXISTS quotes(quote TEXT);")
await db.batch_run([db.statement("INSERT INTO quotes VALUES ('hello')")])
```

## Capability table

| Operation | Status | Notes |
|---|---|---|
| Prepare/bind/query rows | Supported | Local verifier initializes D1 and checks seeded queries. |
| `one_as()` dataclass conversion | Supported | Unit tests cover typed row mapping. |
| Migrations, transactions, retries | Not covered | Use Wrangler/native D1 paths or `.raw` until modeled. |
| Deployed D1 verification | Remote-only | Not part of default checks. |


## Testability

Fake only the methods Xampler calls: `prepare()`, statement `bind()`, `all()`, `run()`, and optional binding `batch()`. Tests should assert query shape and Python row conversion, not the full D1 engine.

## Runtime notes

`None` parameters are converted at the Cloudflare boundary with `cfboundary.d1_null`. Keep SQL explicit; Xampler does not try to hide D1 behind an ORM.
