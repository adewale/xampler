# Cloudflare Best Practices Alignment

Last reviewed: 2026-05-01.

Source baseline: Cloudflare docs snapshot including `developers.cloudflare.com` content and the Cloudflare skills references. This table covers the situations where Cloudflare has an explicit Best Practices page or a product page with an explicit best-practices section that maps to an Xampler primitive.

## Alignment table

| Primitive | Example | Cloudflare best-practice guidance | What Xampler now demonstrates | Remaining gap to ideal Tier 1 |
|---|---|---|---|---|
| Workers | most examples | Keep compatibility dates current; use bindings/env for configuration; avoid secrets in source; stream large bodies; handle errors deliberately. | Every Python Worker has a current compatibility date and `python_workers`; examples keep tokens in `env`; R2 streams binary responses; verifier uses `uv run pywrangler dev`. | Shared request/response/error helpers and explicit max-body guards across all routes. |
| R2 | `r2-01` | Use single upload for small objects; multipart for large/resumable uploads; preserve content type; stream large objects; use presigned URLs for temporary access. | Text upload/read, JPEG upload, streamed JPEG download, byte-for-byte verification, typed metadata/options, object handles, multipart API shape. | Runtime multipart verification and presigned URL/Boto3 companion example. |
| D1 | `d1-04-query` | Use prepared statements and parameter binding; initialize local DB; add indexes for frequent predicates; run `PRAGMA optimize`; inspect query plans; retry transient writes. | Verifier initializes local D1; `D1Statement` wraps prepared statements; `/by-author` uses binding; `db_init.sql` creates `idx_quotes_author` and runs `PRAGMA optimize`; `/explain` verifies the index plan. | Retry helper for transient write errors, mutation route, transaction/batch example, remote/deployed verification. |
| Durable Objects | `durable-objects-07-counter`, `durable-objects-15-chatroom`, `websockets-14-stream-consumer` | Use Durable Objects for stateful coordination; model one object per coordination atom; avoid one global bottleneck; handle retryable/overloaded errors; use alarms/WebSockets where appropriate. | Counter routes map names to independent DO instances; verifier checks named-object isolation; `CounterRef` is a typed stub handle; chatroom and stream examples show WebSocket coordination and alarms. | Retry/backoff wrapper that recreates stubs after retryable errors, concurrent increment verifier, hibernation verifier. |
| Vectorize | `vectorize-17-search` | Choose fixed dimensions and distance metric; validate dimensions; use metadata/namespaces carefully; batch writes; query with `topK`, filters, `returnValues`; use `queryById` where appropriate. | Dataclasses model vectors, metadata, namespace, query options, typed matches/results, and `return_values`; route supports upsert/query/describe. | Deterministic runtime verifier, dimension validation, batch helper, `query_by_id`, documented index creation. |
| R2 SQL | `r2-sql-21-query` | R2 SQL is read-only and single-table; avoid unsupported joins/subqueries; select specific columns; add time filters and `LIMIT`; use approximate aggregates; use `EXPLAIN`. | Basic client and query dataclass preserve the product boundary. | Safe query presets, read-only validation, default `LIMIT`, `EXPLAIN` helper, unsupported-feature examples, real account verification. |
| Workers Assets | `assets-06-static-assets` | Do not wake compute for static files; route dynamic work deliberately; use asset serving for static content. | Verifier checks `/` is served by Workers Assets and `/api/status` reaches Python only for dynamic work. | Cache/header assertions, SPA fallback and not-found examples. |
| Cron Triggers | `scheduled-08-cron` | Local scheduled events are triggered through `/cdn-cgi/handler/scheduled`; keep scheduled logic testable outside runtime. | `ScheduledEventInfo` and `ScheduledRunResult` make cron handling typed/testable; verifier hits both health route and local scheduled endpoint. | Persist an observable side effect and verify it through storage/log capture. |
| Binary responses / Python packages | `images-12-generation` | Workers guidance favors streaming/binary correctness and bounded memory use. | Verifier checks PNG response signature and `content-type`; code keeps image generation small and deterministic. | Query-validated image options, cache headers, optional R2 output. |
| HTML transformation | `htmlrewriter-11-opengraph` | Workers guidance favors edge transformations without unnecessary origin/app work and safe output handling. | `OpenGraphRewriter` isolates the transformation, uses a dataclass model, escapes metadata, and verifier checks output. | Replace explicit string transform with a true Python-native `HTMLRewriter` wrapper when available. |
| Queues | `queues-16-producer-consumer` | Queue handlers should ack or retry each message independently; retries should use backoff; producers should send typed messages. | `QueueJob`, `QueueSendOptions`, `QueueMessage`, and `QueueBatchResult`; producer route is verified; consumer code explicitly ack/retries with capped exponential backoff. | Local consumer harness that verifies batch delivery, ack, retry, and dead-letter behavior. |

## Tier-1 readiness notes

Tier 1 means the example is Pythonic, grounded in Cloudflare vocabulary, and has meaningful verification. For local-only primitives, that means verifier coverage. For account-backed products such as Vectorize and R2 SQL, true Tier 1 still needs either remote-binding verification or a deterministic local substitute that exercises the same API contract.
